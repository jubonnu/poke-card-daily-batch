# テーブル定義（日本語）

Pokemon Price Tracker バッチで使用する Supabase（PostgreSQL）のテーブル一覧です。

---

## 1. sets（セット）

ポケモンTCGの**セット**（弾）のマスタ。API の `GET /sets` から取得。

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | UUID | 主キー（自動採番） |
| tcg_player_id | TEXT | TCGPlayer のセットID（一意） |
| name | TEXT | セット名 |
| series | TEXT | シリーズ名（例: スカーレット&バイオレット） |
| release_date | DATE | リリース日 |
| card_count | INTEGER | カード枚数 |
| image_cdn_url | TEXT | セット画像URL（800x800） |
| image_cdn_url_200 | TEXT | セット画像URL（200x200） |
| image_cdn_url_400 | TEXT | セット画像URL（400x400） |
| image_cdn_url_800 | TEXT | セット画像URL（800x800） |
| price_guide_url | TEXT | TCGPlayer 価格ガイドURL |
| has_price_guide | BOOLEAN | 価格ガイドの有無 |
| no_price_guide_reason | TEXT | 価格ガイドがない場合の理由 |
| created_at | TIMESTAMPTZ | 作成日時 |
| updated_at | TIMESTAMPTZ | 更新日時 |

## 3. card_prices（カード価格・日次スナップショット）

カードの**その時点の市場価格**を日付ごとに1件保存。バッチ実行日の価格を記録。**USD と JPY の両方**を保持（保存時の為替レートで円換算）。

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | UUID | 主キー（自動採番） |
| card_tcg_player_id | TEXT | カードの TCGPlayer ID |
| tcg_player_id | TEXT | 同上（参照用） |
| market_price | NUMERIC | 市場価格（USD） |
| low_price | NUMERIC | 最安価格（USD） |
| market_price_jpy | NUMERIC | 市場価格（JPY・保存時換算） |
| low_price_jpy | NUMERIC | 最安価格（JPY・保存時換算） |
| listings | INTEGER | 出品数 |
| sellers | INTEGER | 出品者数 |
| primary_condition | TEXT | 主な状態（Near Mint など） |
| primary_printing | TEXT | 主な版（Holofoil など） |
| price_date | DATE | 価格の日付 |
| last_updated_at | TIMESTAMPTZ | API 上の最終更新日時 |
| created_at | TIMESTAMPTZ | 作成日時 |

**一意制約**: (card_tcg_player_id, price_date) — 1カード1日1行。

## 為替（USD → JPY）について

価格系テーブルは **USD をそのまま保存**し、バッチ実行時の為替レートで**日本円（JPY）に換算した値も同時に保存**しています。

- **レート**: 環境変数 `USD_JPY_RATE` で指定（未設定時は 200）。バッチ実行時に固定レートで換算。
- **換算タイミング**: DB へ保存する直前。API から取得した USD をそのまま書き、同じレートで JPY を計算して `*_jpy` カラムに格納します。

## 9. batch_runs（バッチ実行ログ）

バッチの**実行結果**を記録。冪等性の確認や監視に利用。

| カラム名 | 型 | 説明 |
|----------|-----|------|
| id | UUID | 主キー（自動採番） |
| batch_type | TEXT | バッチ種別（sets / cards / sealed / full） |
| status | TEXT | 状態（running / completed / failed） |
| started_at | TIMESTAMPTZ | 開始日時 |
| completed_at | TIMESTAMPTZ | 終了日時 |
| cards_fetched | INTEGER | 取得したカード数 |
| sets_fetched | INTEGER | 取得したセット数 |
| prices_updated | INTEGER | 更新した価格件数 |
| credits_used | INTEGER | 消費したAPIクレジット |
| error_message | TEXT | エラー時メッセージ |
| metadata | JSONB | その他（履歴件数・eBay件数など） |

---

## 10. batch_checkpoints（バッチ再開用チェックポイント）

**cards** / **sealed** バッチが途中終了した場合、最後に成功したセット ID を記録する。再実行時はその次のセットから処理する。

| カラム名 | 型 | 説明 |
|----------|-----|------|
| batch_type | TEXT | バッチ種別（**cards** / **sealed**）。主キー |
| last_set_tcg_player_id | TEXT | 最後に処理したセットの TCGPlayer ID |
| updated_at | TIMESTAMPTZ | 更新日時 |

- 全セット処理が完了するとレコードは削除され、次回は先頭から実行される。
- 環境変数 `BATCH_FULL_RUN=true` で実行するとチェックポイントを無視し先頭から実行し、完了時にクリアする。

## テーブル関係のイメージ

```
sets
  └── cards (set_tcg_player_id で紐づく)
        ├── card_prices（日次スナップショット）
        ├── card_price_history（TCG価格履歴）
        ├── card_ebay_prices（PSA等現在価格）
        └── card_ebay_price_history（PSA等価格履歴）

sets
  └── sealed_products (set_tcg_player_id で紐づく)
        └── sealed_product_price_history（価格履歴）

batch_runs（独立・実行ログ）
batch_checkpoints（独立・cards/sealed の再開用）
```
